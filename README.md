# Система учета клиентов такси (Taxi Accounting System)

## Описание проекта
Учебный курсовой проект, демонстрирующий проектирование реляционной базы данных и разработку прикладного модуля на языке Python для автоматизации учета в сфере пассажирских перевозок. Система предназначена для службы такси и решает задачи ведения реестра клиентов, учета парка автомобилей и водителей, обработки заказов, регистрации платежей и формирования аналитической отчетности.

## Структура базы данных
База данных `taxi_db` состоит из 6 связанных таблиц (приведена к 3НФ):
- `clients` — реестр клиентов;
- `drivers` — водители и их рейтинги;
- `cars` — парк автомобилей;
- `orders` — история заказов и поездок;
- `payments` — финансовые транзакции;
- `reviews` — отзывы и оценки клиентов.

## Функциональные объекты БД
В рамках проекта в MySQL реализованы следующие программные объекты:
- **Хранимая процедура:** `create_order` (создание заказа с проверкой статуса водителя и обработкой исключений).
- **Триггер:** `trg_after_review_insert` (автоматический пересчет среднего рейтинга водителя при добавлении отзыва).
- **Представление (View):** `v_order_summary` (сводная аналитика по заказам).
- **Пользовательская функция:** `calculate_fare` (расчет стоимости поездки по расстоянию).
- **Обработчик исключений:** `DECLARE EXIT HANDLER FOR SQLEXCEPTION` внутри процедуры `create_order`.
- **Роли/Пользователи:** `taxi_admin` (полный доступ), `taxi_manager` (диспетчер), `taxi_driver` (только чтение заказов).

## Технические требования
- **СУБД:** MySQL 8.0 или выше (с поддержкой `utf8mb4` и `CHECK` ограничений).
- **Язык программирования:** Python 3.8+.
- **Зависимости:** `mysql-connector-python`.
